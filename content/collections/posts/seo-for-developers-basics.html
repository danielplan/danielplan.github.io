There are many different ways to optimize your website for search engines. In this article, I will cover the more technical aspects of SEO. I will show you the most important HTML Tags and how to use them. Generally, the first thing that a search engine will do after it crawls your page is to analyze the HTML tags and content of your page. Therefore, there are a lot of tags to know, which help you with your SEO. Let's take a look at the most important HTML tags.
This is by no means a complete list of everything you need to know about SEO, but I will keep this updated while I learn more about this topic, so that it can be used as a reference for you.

The -tag: Meta tags

The most widely known way to improve the SEO of a website is to add meta tags to the head of the page. Meta tags contain additional information about the web page and about the content it displays. Such tags are for example:

  • Title
  • Description
  • Keywords
  • etc.
Let's have a look at them in detail and see how they can and how they are used.

Title

The title tag is the most important meta information of your page. It is a short text which gives a brief headline to the content of the page. It is displayed as the title of the search results and also showed in the browser tab. Using it is very easy and looks something like this: 


    
        
        A fascinating title | daniel plan.

Robots meta tag

The robots meta tag is used to explain to the crawler (the robot) which pages can be indexed and which pages can't, as well as if the links of the page should be followed (crawled) by the crawler. For that, we need to specify if the page is indexable (index) or not indexable (noindex). In addition to that we need to specify if the page is followable (follow) or not followable (nofollow). This may look something like this: 



    
        A fascinating title | daniel plan.
        
        


    
My website

The values specified above (index/noindex, follow/nofollow) are given with the structure above, comma seperated in the content attribute.

Canonical meta tag

The canonical meta tag is used to describe that the page is a duplicate of another page. This may be handy if you have multiple pages with the same content but on different paths of your website. For instance, you might run into something like this:

You have a page that displays a product on the path /products/cool-shirt/. However you also want to make it possible to access the page with the path /summer-line/cool-shirt/. For such cases you need to use the canonical meta tag to tell the crawler that the page is a duplicate of the page with the path /summer-line/cool-shirt/.

Open Graph meta tags

The last meta tags I want to get into are the Open Graph meta tags. These are similar to the title and description meta tags, but they are not used by search engines, but by social media sites. They are used to describe the page to social media sites, so that a small widget can appear if someone shares your website on social media platforms.

This allows you to add an image, a title and a description, as well as a type, a site name and a url, which are then displayed in the social media widget in posts.

Structured data: Schema.org

Another important topic for SEO is structured data. Similar to meta data, structured data is used to give the search engines additional information about content of a page, so that they can render beautiful and understandable widgets to the user of the search engine, so-called "rich results". This data is not only used by search engines, but also other agents, who crawl the web.

In this section, I will only give a brief overview of some types and how it is used. For more information, please consult the Schema.org-website.

Types of Schemas

Now that you have a simple understanding on how to use structured data, let's have a look at some more types of structured data. There are many different types that are available for structured data, the most common ones are for:

  • Organization: used to describe information about an organization such as companies, schools etc.
  • Person: This type is used to describe a person.
  • Product: This describes a product or a service, it contains information such as price, rating etc.
  • Event: This type is used to describe an event, it contains information such as date, location etc. Events may be lectures, concerts, festivals, workshops etc.
  • Recipe: Used to describe a recipe, it contains information such as ingredients, cooking instructions etc. Recipes are rendered as widgets on the search engine's result page. Using the data of the schema, recipes can even be found using ingredients and calories.
  • Place: This type is used to describe a place, it contains information such as address, opening hours etc. It is used to describe a location, such as a restaurant, a hotel, a museum etc.

If you are interested in more types of structured data, Google offers a great resource where they list the different types and how they are rendered on the search engine's result here.
